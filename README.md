# CalendAria

This is repository for simple calendar app that I wanted to create mainly for myself so I can structure my day and week toghether with tracking my progress for repetitive tasks. 

Since before this project I haven't coded in kotlin and since I have started university I have used more hardware-related languages (like C, Cpp, SystemVerilog, Assembly), I have decided to try out AI in creation of this app. Also - the size and amount of functions I wanted to implement, making it alone and that I wanted to make it as quickly as possible infuenced that decision. 

## AI usage

I have not tracked it anywhere but half of the project was coming up with external AI - that did not have any knowledge about my file structure other than that I have specified for it so it was a bit slower with much more errors along the way. So sometimes (more with common sense about coding rather than kotlin knowledge) I had to fix some issues myself since AI was stuck on the problem and did basically loops of bad, not working solutions. 

The AI inside the Android Panda was much better and quicker with solutions, also it might have followed the folder structure I have created for the project when creating new functionalities - or I might have specified it when writing prompts; I don't remember that's why I'm saing "might" :)
